Decisão Estratégica: PT Only

Escopo restrito a Principal Tokens (PT) — yield fixo garantido até maturidade. YT excluído para capital de usuários por incompatibilidade com a proposta DollarUp de "rendimento passivo sem risco de perda do principal".

YT pode ser considerado futuramente apenas para treasury própria da DollarUp (fees acumulados), nunca para capital de usuários.
01

Objetivo do Sistema e Modos de Operação

O objetivo primário é monitorar, classificar e rebalancear de forma autônoma alocações em PT do protocolo Pendle Finance na Arbitrum, mitigando sistematicamente riscos de cauda, slippage e deterioração de liquidez.

Para eliminar ambiguidades na tradução de regras de negócio para código, o sistema obedece às seguintes interfaces:

2.1 Interface do Agente Principal (Core Agent)

class PendleAllocationAgent:

    def observe(self) -> MarketSnapshot:
        """Coleta dados on-chain da AMM Pendle, taxas de gás e oráculos."""

    def pre_validate(self, snapshot: MarketSnapshot, ctx: AssetContext) -> PreValidationResult:
        """Aplica filtros: TVL mínimo, Fixed APY mínimo, DaysToMaturity, limites de pool."""

    def execute(self, decision: ExecutionDecision) -> ExecutionResult:
        """DRY_RUN: simula. LIVE: transmite via safe_executor após Slippage Gate."""

    def cooldown(self, reason: str, duration: int) -> None:
        """Aplica estado de espera conforme matriz de falhas."""

    def handle_failure(self, error: AgentError) -> None:
        """Incrementa contador horário e avalia gatilhos de escalonamento de risco."""

2.2 Funções do Core Quantitativo (Quant Math Engine)

def compute_delta_net(ctx: AssetContext, snapshot: MarketSnapshot) -> float:
    """ΔNet anualizado deduzindo CTEA completo (WAD precision)."""

def compute_dynamic_threshold(ctx: AssetContext, volatility_24h: float) -> float:
    """Δtarget_effective = Δtarget_base + (k × σ_24h)"""

def should_enter_pt(delta_net: float, threshold: float, regime_bias: float) -> bool:
    """ΔNet < -(threshold × regime_bias) → PT atrativo (mercado superprecifica yield)."""

def should_exit_position(pos: PositionState, snapshot: MarketSnapshot, ctx: AssetContext) -> bool:
    """Gatilhos protetivos obrigatórios:
    1. Quebra de Peg / risco do subjacente (LST/LRT descolando).
    2. Fixed APY colapsou abaixo do piso histórico de segurança.
    3. Deterioração severa de TVL e liquidez interna da AMM."""

3.1 Limites de Alocação por Tese e Pool

Perfil AGGRESSIVE removido — incompatível com PT-Only para capital de usuários DollarUp.

Lógica Quantitativa de Decisão e Consistência Dimensional

Custos pontuais (Gas e Slippage) são anualizados dividindo pelo prazo de vencimento. Toda aritmética usa precisão WAD.

4.1 CTEA Anualizado

CTEA = (GasUSD + SlippageInUSD + SlippageOutUSD) / NotionalUSD × (365 / DaysToMaturity)

4.2 ΔNet

ΔNet = ΔTime − CTEA − RiscoSaídaAnualizado
ΔTime = fixed_apy − implied_apy
RiscoSaída = 0.005 (0.5% a.a. conservador)

4.3 YVI — Yield Volatility Index (σ24h)

Definido como o desvio padrão amostral dos valores de ΔNet calculados a cada hora nas últimas 24 horas. Baliza o limiar dinâmico:

Δtarget_effective = Δtarget_base + (k × σ24h)

4.4 Árvore de Decisão PT (PT-Only)

# Prioridade 1 — Saída protetiva (verifica antes de qualquer entrada)
if fixed_apy < min_fixed_apy:              → EXIT_PT  # APY colapsou
if tvl_usd < min_tvl × 0.3:               → EXIT_PT  # TVL deteriorou
if days_to_maturity < 7 and position > 0: → EXIT_PT  # Vencimento próximo

# Prioridade 2 — Entrada: mercado superprecifica yield → PT atrativo
if delta_net < -(threshold × regime_bias):  → ROTATE_TO_PT

# Prioridade 3 — Inércia: banda efetiva
if -threshold ≤ delta_net ≤ threshold:      → HOLD_NO_OP
Nota: YT Decay Gate (BreakevenDays) removido do escopo PT-Only. BREAKEVEN_VIOLATION não aplicável.

4.5 CES — Capital Efficiency Score (PT)

CES = ΔNet / CTEA
Usado para rankear pools concorrentes. Pool com maior CES recebe alocação preferencial.

Camada Inteligente e Fallback Preditivo

O classificador AI atua estritamente como modificador de viés das equações matemáticas. Se confiança < 65%, o motor ativa o Fallback Determinístico, forçando regime_bias = 1.0.

Claude — Classificador de Regime

  • Modelo: claude-sonnet-4-20250514
  • Input: snapshot JSON (fixed_apy, implied_apy, ΔNet, TVL, DTM, YVI)
  • Output: regime, confidence, regime_bias, reasoning
  • Threshold: PREDICTOR_CONFIDENCE_THRESHOLD = 0.65
  • Remove markdown automático (Claude retorna com backticks)
  • Fallback se confidence < 0.65 ou erro de API

7.1 Protocolo de Saída de CRITICAL_HALT

Uma vez em travamento, o agente interrompe assinaturas e TXs on-chain, mas mantém em background as rotinas observe() e pre_validate() de forma passiva para telemetria.


Não há saída automática via cron ou scripts. O destravamento exige intervenção manual do operador: setar PENDLE_AGENT_HALT=false no .env e reiniciar o worker.

Governança de Telemetria, Retenção e Reprodutibilidade

Todos os payloads são estruturados em JSON e persistidos em pendle_telemetry.jsonl com retenção mínima de 12-24 meses para auditoria regulatória e tuning de modelos.

Garantia de Atomicidade: Em modo LIVE, o payload completo é gravado ANTES da TX ser submetida. O replay_id contém contexto exato e imutável da decisão — mesmo em crash do nó, a decisão é auditável.
